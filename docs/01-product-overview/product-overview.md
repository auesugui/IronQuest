# Product Overview

> Source: PRD v2.0 Sections 1–2

---

## One-Line Pitch

**Iron Quest is a workout tracker where every rep you log earns currency to raise, feed, and strengthen a digital pet that battles its way up an endless tower.**

---

## The Core Loop

The app has two distinct modes that feed into each other:

| Phase | Name | Where | What Happens |
|-------|------|-------|--------------|
| **A** | **Earn** | The Gym | Player logs sets/reps → earns Forge Points (FP) based on volume, consistency, PRs |
| **B** | **Spend** | The Den | Player feeds pet, allocates stats, trains pet, battles the tower |

> **KEY DESIGN RULE:** The workout tracker must function perfectly as a standalone tool. If the pet system disappeared tomorrow, the tracker should still be worth using.

---

## Design Pillars

| Pillar | Meaning |
|--------|---------|
| **Tracker First, Game Second** | Workout logging is the highest priority. The game layer is a reward, not a requirement |
| **Earn Everywhere, Spend Intentionally** | Points flow automatically from workouts. Spending requires deliberate choices |
| **Your Pet = Your Effort** | The pet is a living reflection of training history. No shortcuts |
| **Geometric Identity** | Pets are abstract geometric SVG creatures shaped by stat distribution |

---

## Competitive Positioning

Iron Quest sits at the intersection of:
- **Workout trackers** (Strong, Hevy, JEFIT) — serious rep logging
- **Virtual pet games** (VoidPets, Tamagotchi) — creature care and attachment
- **Game RPGs** (Pokemon, Fire Emblem) — battle tower, type system, evolution

No current app combines a serious workout tracker with a creature-raising battle game where pet growth is directly tied to real training data.

### Inspiration References

| Reference | What It Inspires |
|-----------|-----------------|
| VoidPets | Aesthetic, creature attachment, clean mobile UX |
| Tamagotchi | Care/feeding loop, emotional bond with digital creature |
| Pokemon | Battle tower, type system, evolution excitement |
| Undertale | Geometric character design that conveys personality without realistic art |

## Target User

### Primary Persona

- **Age:** 25–40
- **Experience:** Intermediate lifter
- **Program:** Structured (PPL, Upper/Lower, Bro Split)
- **Frequency:** 4–6 days/week
- **Pain points:** Consistency during deloads, busy periods, motivation dips
- **Interests:** Gaming, progression systems, collectibles, visual feedback

### User Needs

| Need | Metric |
|------|--------|
| **Friction-free logging** | Rep input under 3 seconds between sets |
| **Visible progress** | Pet growth, stat rises, tower floors cleared |
| **Meaningful choices** | FP allocation creates engagement beyond logging |
| **Motivation on bad days** | Pet hunger + streak provide external reason to show up |
| **Fun without pressure** | Battle tower is optional, no harsh punishment for missing a day |
