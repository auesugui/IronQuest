# Implementation Priority

> Source: Tracker Spec §8

---

## Build Order

Aligned with PRD roadmap phases. Not all systems need to ship in Phase 1.

| Feature | Phase | Priority | Notes |
|---------|-------|----------|-------|
| Core session flow (exercise list, rep logging, rest timer) | 1 | **P0** | Absolute foundation |
| Weight memory + auto-fill from last session | 1 | **P0** | Essential for speed |
| Template browser (4 core: PPL, UL, Full Body, Minimalist) | 1 | **P0** | Needed for onboarding |
| FP calculation engine (base + volume + streak) | 1 | **P0** | Core economy |
| Post-session summary with FP breakdown | 1 | **P0** | Critical motivation moment |
| Loadout screen with Session Intent (Normal + Deload only) | 1 | **P1** | Start with 2 intents, expand later |
| Copy & Customize template editing | 1 | **P1** | Power users need early |
| Personal Baseline + relative FP scaling | 1 | **P1** | During first 3 sessions. Needed for fairness |
| FP distribution radar chart on templates | 1 | **P1** | Strategic template selection. Simple bar chart first OK |
| Full Session Intent suite (Tempo, Pause, Drop Set, Rest-Pause) | 2 | **P1** | Adds depth after core loop proven |
| Cardio session logging (all 4 types) | 2 | **P1** | LISS first, then HIIT/Hybrid |
| Superset & circuit support | 2 | **P2** | Nice-to-have |
| Weekly quest system | 2 | **P2** | Adds variety after loop established |
| Build-from-scratch program editor | 2 | **P2** | Copy & Customize covers most cases |
| Auto-tagging engine for custom exercises | 2 | **P2** | Manual tagging works for MVP |
| Progressive overload suggestions (gold arrow) | 3 | **P2** | Needs several weeks of data first |
| Expanded template library (8–10 total) | 3 | **P2** | 4 templates covers MVP |

### Priority Legend

| Level | Meaning |
|-------|---------|
| **P0** | Must ship in Phase 1 |
| **P1** | Should ship in target phase, critical for experience |
| **P2** | Nice-to-have, can slip without blocking launch |
